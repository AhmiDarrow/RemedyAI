# Explain Code

## When to use
"what does this do", "walk me through X", onboarding into a module.

## Steps
1. Read the target file(s) with tools — don't invent.
2. Explain purpose, inputs/outputs, and control flow.
3. Call out non-obvious constraints, side effects, and dependencies.
4. Match depth to the user (high-level first, then details if needed).
